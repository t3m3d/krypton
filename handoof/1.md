# macOS Handoff: Choc / Objective-K Parity

Windows agent: Forge
Date: 2026-07-07

## Goal

Make Objective-K / Choc feel real on both macOS and Windows without adding C.

Public direction:
- Use `Choc` as the shared native UI wrapper name.
- Keep Cocoa as the macOS backend concept if useful.
- Do not name Windows APIs Cocoa.
- Prefer Krypton / KryptScript.

## Windows Findings

Current Windows stack:
- `stdlib/choc.k` routes to `k:gui`.
- `stdlib/objkwin.k` routes Obj-K-shaped `ok*` calls through `k:choc`.

What works:
- Plain Windows GUI works.
- Choc standalone window works.
- Direct Obj-K facade to GUI worked in isolation.
- `chocWindow()` needed handle coercion:

```k
func chocWindow(app, title, w, h) {
    let win = guiWindow(title, w, h)
    emit win + ""
}
```

Without `win + ""`, wrapped window handles could crash when returned through another module layer.

What still needs hardening:
- `objkwin -> choc -> gui` still needs another wrapper-boundary pass.
- Next likely Windows fix is applying the same coercion at `okWindow()`:

```k
func okWindow(app, title, w, h) {
    let win = chocWindow(app, title, w, h)
    emit win + ""
}
```

## What macOS Should Check

Please verify the same class of bug on macOS:
- Object/handle lifetime across wrapper layers.
- Cocoa object refs returned through Choc or Obj-K wrappers.
- Whether macOS wrapper returns need explicit string/object coercion before crossing module boundaries.
- Whether a thin `choc.k` over `cocoa.k` is safe.

Suggested macOS tests:
- Tiny Choc window opens and stays alive.
- Obj-K wrapper window opens and stays alive.
- Obj-K wrapper with label/button/text field opens and stays alive.
- Event callback receives sender-shaped args consistently.
- App exits cleanly.

## Compatibility Notes

Do not break:
- Existing `stdlib/cocoa.k` behavior.
- Existing macOS Obj-K app examples.
- Current kweb GUI work.

Coordinate naming:
- Shared public layer: `Choc`.
- macOS backend: Cocoa is fine internally.
- Windows backend: Choc over `gui`.

## One-Line Status

Windows has Choc mostly working, but Obj-K through Choc still needs wrapper-boundary hardening; macOS should check object refs across the same facade layers.
